# 发货确认收入 - 导入与同步记录（原型初稿）

## 原型任务单

| 项目 | 说明 |
|---|---|
| 页面 | 发货确认收入列表页 |
| 主任务 | 导入账单/费用数据，并查看人工上传与系统同步的处理记录 |
| 明确要求 | 平台、店铺、月份（下拉单选）、文件类型；主体公司自动回显；按唯一键去重；原文件和错误明细可下载；系统同步记录可查询下载 |
| 明确不做 | 覆盖导入、人工修改主体公司、审批流、页面内角色模拟 |
| 保守假设 | 文件类型、字段模板和唯一键由后端配置；上传记录至少保存原始文件、解析结果和错误明细 |

## 页面来源映射

| 可见元素 | 来源 | 必需 |
|---|---|---|
| 财务系统壳层、发货确认收入页签、筛选和结果表格 | 用户提供的现有页面截图 | 是 |
| 导入数据按钮 | 用户明确需求 | 是 |
| 导入弹窗中的平台、店铺、月份、文件类型和主体公司 | 用户明确需求 | 是 |
| 新增模式说明、重复跳过结果 | 用户确认规则 | 是 |
| 导入/同步记录抽屉、原文件下载、错误明细下载 | 用户明确需求 | 是 |
| 系统同步来源和同步下载 | 用户确认的 Python 爬取 + Java 导入流程 | 是 |

## UI 设计契约

- 以 `tokens.css` 为唯一颜色、间距和圆角来源。
- 平台、店铺、文件类型使用 Ant 风格 Select 触发器；不使用可见原生 select。
- 月份使用 Ant 风格的下拉单选，默认显示当前查询月份。
- 仅支持新增、不允许覆盖是后端业务规则，不在导入表单中单列展示。
- 状态标签必须使用 `.tag` 与对应 `.tag--*`。
- 查看导入记录使用右侧抽屉，不跳转新页面。
